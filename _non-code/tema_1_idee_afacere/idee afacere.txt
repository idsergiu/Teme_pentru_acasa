Tip prompt:
Meta Prompt + Chain-of-Thought

Titlu: Chatbot generic AI pentru Support Center

Prommpt:
-------------------------------------------------------------------------------
Rol: Ești un expert în startup-uri AI și arhitectură de sisteme bazate pe LLM.

Sarcină:
Analizează și dezvoltă următoarea idee de afacere:

"Idee: chatbot AI generic pentru Support Center, care poate fi specializat pe baza de skilluri pentru diferite industrii (e-commerce, telecom, banking etc.) și care poate memora istoricul fiecărui client, inclusiv problemele anterioare și soluțiile oferite. Acorda importanta notiunii de generic, pentru ca specializarea pe industrie (e-commerce, aviatie) se va face cu fisiere md de skilluri specifice si exhaustive. 

Instrucțiuni:
1. Analizează problema pe care o rezolvă acest produs.
2. Descrie cum ar funcționa chatbotul din punct de vedere tehnic.
3. Explică cum pot fi implementate skillurile specializate.
4. Explică cum poate fi implementată memoria clienților (customer memory).
5. Propune funcționalitățile principale ale produsului.
6. Sugerează un model de business și monetizare.

Formatul răspunsului:

1. Problema rezolvată
2. Soluția propusă
3. Arhitectura tehnică
4. Sistemul de skilluri
5. Memoria clienților
6. Funcționalități cheie
7. Model de business
8. Avantaje competitive
9. Posibile riscuri


===================

Raspuns ChatGpt 5.2
----------------------------------------------------
1. Problema rezolvată

Majoritatea companiilor au Support Center-uri costisitoare și ineficiente, cu câteva probleme majore:

1. Costuri operaționale mari

agenți umani scumpi

training constant

rotație mare de personal

2. Knowledge fragmentation

Informația este răspândită în:

documentație

wiki intern

CRM

emailuri

SOP-uri

Rezultatul: agenții nu găsesc rapid răspunsurile corecte.

3. Chatboți existenți sunt slabi

Majoritatea sunt:

rule-based

FAQ search

sau LLM-uri fără memorie

Probleme:

nu înțeleg contextul

nu țin minte clientul

nu oferă rezoluții complete

4. Lipsa memoriei istorice a clientului

De fiecare dată clientul trebuie să repete:

problema

produsul

istoricul

Aceasta produce frustrare și timp pierdut.

5. Dificultate în adaptarea la industrie

Fiecare industrie are:

reguli

procese

termeni

Exemple:

e-commerce → retururi

telecom → abonamente

banking → KYC / securitate

Un chatbot generic nu știe aceste procese.

2. Soluția propusă

Un AI Support Platform compus din:

1️⃣ Core chatbot generic

Un LLM orchestrat care știe:

conversație naturală

rezolvare de probleme

workflow support

Acesta este agnostic de industrie.

2️⃣ Skill system

Specializarea se face prin:

skill packs (.md files)

care conțin:

procese

reguli

fluxuri

knowledge

Exemple:

skills/
   ecommerce/
      returns.md
      refunds.md
      shipping.md

   telecom/
      subscriptions.md
      billing.md
      sim_activation.md
3️⃣ Customer Memory System

Un sistem care reține:

istoricul clientului

probleme anterioare

produse cumpărate

soluții oferite

Chatbotul poate spune:

"Ultima dată ai avut o problemă cu livrarea comenzii #34892. Vrei să verificăm dacă s-a rezolvat?"

4️⃣ Orchestrator AI

Un layer inteligent care decide:

ce skill folosește

ce memorie recuperează

ce tool execută

4. Sistemul de skilluri

Skillurile sunt module de comportament AI.

Format: .md

Exemplu:

# Skill: Product Returns

## Description
Handle product return requests for e-commerce customers.

## Steps

1. Verify order number
2. Check return eligibility (30 days)
3. Ask reason
4. Offer refund or replacement

## Policies

- Electronics: 14 days
- Clothing: 30 days

## Tone
Helpful and solution-oriented
Cum sunt folosite

Orchestratorul detectează:

intent: return product

și încarcă:

returns.md

INDUSTRY SKILLS
↓
COMPANY SKILLS

   customer_empathy.md

   ecommerce_returns.md

   amazon_return_policy.md
5. Memoria clienților

Customer memory este esențială pentru suport real.

Tipuri de memorie
1️⃣ Profile memory

Date persistente:

customer_id
name
email
language
loyalty_status
2️⃣ Interaction memory

Evenimente:

ticket_opened
refund_issued
complaint
3️⃣ Problem memory

Probleme anterioare:

problem: late delivery
solution: refund 10%
status: resolved
4️⃣ Preference memory

Preferințe:

prefers chat
hates phone calls
VIP customer
Implementare

Storage:

Postgres / DynamoDB
+
Vector DB
Retrieval

La fiecare conversație:

fetch customer memory
summarize
inject in prompt

Exemplu:

Customer summary:
- VIP customer
- 3 orders last month
- Previous issue: delayed shipping

3️⃣ Compliance

banking

4️⃣ Costuri LLM
